# Task: 010-tool-processor

**Global References:** 
- `rfc-001.implementation-guide.md` (MUST READ for Strict Timeouts and Atomic Events)
- `rfc-001.revision-06.md` (Section 6.3 Tool Safety & Hanging Prevention)

## 1. Objective & Responsibility
Implement the `ToolProcessor` (Motor Cortex). It is the exclusive node in the system allowed to perform physical Operating System Shell Commands on behalf of the Agent logic. It translates `EFFECT_RUN_TOOL` intents into safe `OSAdapter` executions, strictly enforcing "Zombie-Proof" environments.

## 2. Invariants & Global Contracts (The Happy Path)
- **The Zombie Prevention Invariant:** A classic agent failure is executing `npm start` or an interactive SSH command, causing the CLI to hang infinitely waiting for stdin. The ToolProcessor MUST spawn every process strictly with `.exec()` parameters that drop/ignore stdin.
- **The Atomic Log Invariant:** You MUST internally buffer all chronological `stdout` and `stderr` generated by the OS. You are strictly forbidden from emitting a durable event line-by-line. Instead, you MUST group all generated logs into a single aggregated `TOOL_COMPLETED` event once the OS exit code is definitively captured. This guarantees the Ledger accurately reflects atomic unit-of-work state.

## 3. I/O Boundaries & Dependencies
- **Strict Adapter Injection:** You MUST inject `OSAdapter`. You MUST NOT import `execa`, `child_process`, or `shelljs`. Do not cheat the interface.
- **Runtime Overrides:** The Hub configuration may dictate physical constraints like `cwd` (Current Working Directory). The processor must pull this exact `cwd` and pass it through to the `OSAdapter.exec` interface.

## 4. Exact Event Signatures (Contract Adherence)
- **Listens To:** `EFFECT_RUN_TOOL` (Must parse `command: string` and `args: string[]`).
- **Yields (Volatile):** `TOOL_STDOUT_CHUNK` (for progressive UI visualization).
- **Yields (Durable):** `TOOL_COMPLETED` (Successful exit code 0) or `TOOL_FAILED` (Non-zero exit code or `timeoutMs` breach). Both must include the absolute, aggregated `stdout/stderr` string.

## 5. Lifecycle & Unhappy Path
- **Muted Hydration Mode:** If an incoming `EFFECT_RUN_TOOL` carries the `isReplay: true` flag, you MUST silently drop it. During boot hydration, we cannot re-run `rm -rf` or `git commit`. 
- **Global Panic Interrupt:** If `CIRCUIT_INTERRUPTED` arrives on the stream, the ToolProcessor MUST immediately locate all active running child processes (via tracking IDs mapped to the `OSAdapter` handles) and forcefully dispatch `SIGKILL`/`SIGTERM` to them to prevent orphaned bash instances.

## 6. Required Execution Constraints
- Implement a 60000ms (60 second) hard-coded standard timeout. When invoking `this.osAdapter.exec()`, always pass this timeout limit.
- If the `OSAdapter` throws a timeout error, translate that physical error explicitly into the `TOOL_FAILED` event payload with a specific `reason: "TIMEOUT"`.
- Keep an internal Map or tracker of currently executing Process Promises linked to their `eventId`.

## 7. Definition of Done
1. **The Hydration Pass-through Proof:** Yield an `EFFECT_RUN_TOOL` event containing `isReplay: true`. Assert that the `OSAdapter` was invoked exactly 0 times.
2. **The Atomic Aggregate Proof:** Use a mock `OSAdapter` that returns three strings: "A", "B", "C". Assert that the ToolProcessor yielded exactly one `TOOL_COMPLETED` durable event, and that its payload's log exactly equals "ABC".
